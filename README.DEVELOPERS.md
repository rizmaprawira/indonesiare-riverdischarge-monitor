## Architecture Overview

### System Design Pattern

```
┌─────────────────────────────────────────────────────────────────┐
│                    FRONTEND LAYER (Browser)                     │
│  React 18 + TypeScript + Leaflet (react-leaflet)                │
│  ├─ MapView: Tile layers, point markers, hover interaction      │
│  ├─ SummaryPanel: River rankings, point selection               │

                        │ fetch public/data/latest/
┌───────────────────────▼─────────────────────────────────────────┐
│                   STATIC DATA LAYER (CDN/GitHub Pages)          │
│  Pre-computed JSON and PNG tiles (no database needed)           │
│  ├─ tiles/{z}/{x}/{y}.png - Web Mercator PNG tiles (z 4-8)     │
│  ├─ metadata.json - Latest date, timestamps, available dates   │
│  ├─ layers.json - Tile URLs for each date                      │
│  ├─ points.json - 600 sampled river stations (lat/lon/name)    │
│  ├─ series.json - Time-series discharge for each point          │
│  ├─ rankings.json - Top 50 rivers ranked by discharge          │
│  ├─ grids.json - Full 24-hour grid values (for hover query)    │
│  └─ boundaries/provinces.geojson - Admin boundaries            │
└───────────────────────┬─────────────────────────────────────────┘
                        │ generated by
┌───────────────────────▼─────────────────────────────────────────┐
│              DATA PROCESSING LAYER (Python Scripts)             │
│  Runs offline to transform raw NetCDF → web-ready assets        │
│  ├─ process_minimal.py - Main ETL pipeline                      │
│  ├─ generate_tiles_v3.py - Creates Web Mercator PNG tiles       │
│  └─ lib/ - Shared utilities for NcDF processing                │
└───────────────────────┬─────────────────────────────────────────┘
                        │ consumes
┌───────────────────────▼─────────────────────────────────────────┐
│                   DATA SOURCE LAYER                              │
│  NetCDF files from GloFAS (Copernicus)                          │
│  ├─ data_[DATE_RANGE].nc - Daily discharge grids               │
│  │   Variable: dis24 (mean discharge, 24h) in m³/s             │
│  │   Grid: ~1km resolution across Indonesia                    │
│  └─ dimensions: lat × lon × time                               │
└─────────────────────────────────────────────────────────────────┘
```

## Data Pipeline

### Stage 1: Data Acquisition

#### Option A: Manual Download (Current)
```bash
# Download manually from Copernicus CDS
# https://cds.climate.copernicus.eu/
# Place as: data_20-29march2026.nc
```

#### Option B: CDS API (Recommended for Automation)
```python
import cdsapi

client = cdsapi.Client()
client.retrieve(
    'glofas-historical',
    {
        'year': 2026,
        'month': '03',
        'day': ['20', '21', ..., '29'],
        'time': '00:00',
        'variable': 'river_discharge_in_the_last_24_hours',
    },
    'data_20-29march2026.nc'
)
```

### Stage 2: Data Processing (`process_minimal.py`)

**Input**: `data_20-29march2026.nc` (NetCDF 4GB)

**Processing**:
1. **Load NetCDF**: xarray reads lat/lon/time grid
2. **Extract variable**: Pull `dis24` (mean discharge) for all timestamps
3. **Spatial clipping**: Clip to Indonesia bounds (94°E-145°E, 12°S-8°N)
4. **Point sampling**:
   - Load province boundaries from GeoJSON
   - Rank grid cells by mean discharge
   - Java/Sulawesi/Sumatra get bonus weighting
   - Select top 600 points, ensure ≥2 per province
5. **Time-series extraction**: Extract daily values for each point
6. **Ranking computation**: Top 50 rivers per date
7. **Grid serialization**: Save full grid as JSON for hover queries

**Output** (to `public/data/latest/`):
```
metadata.json       # {"latestDate": "2026-03-29", "availableDates": [...], ...}
points.json         # [{id, name, lat, lon, province, basin}, ...]
series.json         # {point_id: {date: discharge, ...}, ...}
rankings.json       # [{date: "2026-03-29", ranking: [{point_id, discharge}, ...]}, ...]
grids.json          # {date: {lat_idx: {lon_idx: discharge, ...}, ...}, ...}
```

### Stage 3: Tile Generation (`generate_tiles_v3.py`)

**Input**: `data_20-29march2026.nc`, style config

**Processing**:
1. **Load color ramp**: From `config/style.config.json`
   ```
   1 m³/s → #F6F9D6 (pale yellow)
   5 m³/s → #E5F2C8
   ... (11 stops total)
   1000 m³/s → #1D3E7B (dark blue)
   ```
2. **For each zoom level** (4, 5, 6, 7, 8):
   - Get Web Mercator bounds for each tile (256×256 px)
   - Extract discharge grid for tile bounds
   - Interpolate to pixel grid
   - **Colorize**: Map discharge → RGBA using color stops
   - **Write PNG**: Use rasterio + Pillow
3. **Output**: `public/data/latest/tiles/{z}/{x}/{y}.png`

**Colorization Logic**:
```python

    # For each value, find which color stop it falls between
    # Use logarithmic interpolation
    # Return RGBA array (semi-transparent)
```

**Why PNG, not GeoTIFF?**
- Smaller file size (web-optimized)
- Faster browser rendering
- Standard Web Mercator tile format
- Opacity support for overlay effects

### Stage 4: Build & Deployment

**Build** (`npm run build`):
1. Clean dist/ directory
2. TypeScript type checking
3. Vite bundles React app
4. Copies `public/data/` to `dist/public/data/`

**Deploy** (`npm run deploy`):

### Automated Daily Updates (GitHub Actions)

**Create** `.github/workflows/update-data.yml`:

```yaml
name: Daily Data Update

on:
  schedule:
    - cron: '0 2 * * *'  # 2 AM UTC daily
  workflow_dispatch:      # Manual trigger

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          token: ${{ secrets.GITHUB_TOKEN }}

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'

      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: '18'

      - name: Install dependencies
        run: |
          pip install -r requirements.txt
          npm ci

      - name: Download latest GloFAS data
        env:
          CDSAPI_KEY: ${{ secrets.CDSAPI_KEY }}
        run: |
          python scripts/download_cds.py

      - name: Process data
        run: |
          python scripts/process_minimal.py
          python scripts/generate_tiles_v3.py

      - name: Build site
        run: npm run build

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist

      - name: Notify completion
        if: always()
        run: echo "Data update complete. Check https://github.com/..."
```

**Create CDS download script** (`scripts/download_cds.py`):

```python
#!/usr/bin/env python3
import cdsapi
from datetime import datetime, timedelta
from pathlib import Path

client = cdsapi.Client()
proj_root = Path(__file__).parent.parent

# Calculate date range (e.g., last 10 days)
today = datetime.utcnow()
start_date = today - timedelta(days=9)

days = [str((start_date + timedelta(days=i)).day).zfill(2) 
        for i in range(10)]
month = str(start_date.month).zfill(2)
year = str(start_date.year)

output_file = proj_root / f"data_{start_date.day}-{today.day}{month}{year}.nc"

print(f"Downloading GloFAS data to {output_file}")

client.retrieve(
    'glofas-historical',
    {
        'year': year,
        'month': month,
        'day': days,
        'time': '00:00',
        'variable': 'river_discharge_in_the_last_24_hours',
    },
    str(output_file)
)

print(f"Downloaded to {output_file}")
```

## Tile Generation

### Understanding Web Mercator Tiles

**Standard format for web maps**:
- **Coordinate system**: EPSG:3857 (Web Mercator)
- **Tile size**: 256×256 pixels at each zoom level
- **Zoom levels**: 0 (world) to 28 (street detail)
- **Tile URL scheme**: `/{z}/{x}/{y}.png`
  - `z`: Zoom level (0-28)
  - `x`: Column (0 to 2^z - 1)
  - `y`: Row (0 to 2^z - 1)

### Tile Generation Process

**For each zoom level** (4, 5, 6, 7, 8):

1. **Calculate tiles covering Indonesia**:
   ```python
   # Using mercantile library
   tiles = list(mercantile.tiles(94.0, -12.0, 145.0, 8.0, zooms=5))
   # Returns: [(z=5, x=22, y=28), (z=5, x=23, y=28), ...]
   ```

2. **For each tile**:
   - Get Web Mercator bounds (lat/lon)
   - Extract discharge grid for bounds
   - Interpolate to 256×256 pixel grid
   - Colorize using color ramp
   - Write PNG

3. **Optimization**:
   - Tiles with all `nodata` are skipped
   - Transparency (alpha=210) allows base map visibility
   - PNG compression reduces file size

### Performance Notes

- **Tile generation time**: ~5-10 minutes per full dataset
- **Output size**: ~200-500 MB (all 5 zoom levels)
- **Served from**: GitHub Pages CDN (cached globally)
- **Browser caching**: Tiles cached aggressively (1 year)

## Troubleshooting

### Issue: Tiles not appearing on map

**Symptoms**: Map shows but no heatmap overlay

**Diagnosis**:
```bash
# 1. Check tile files exist
ls public/data/latest/tiles/*/
# Should show: 4/, 5/, 6/, 7/, 8/

# 2. Check metadata.json has tile URLs
cat public/data/latest/layers.json | jq '.layers[0]'

# 3. Browser console for 404 errors
# Open DevTools → Network tab → look for failed tile requests
```

**Solutions**:
```bash
# Regenerate tiles
python scripts/generate_tiles_v3.py

# Verify path format
# Tiles should be at: public/data/latest/tiles/z/x/y.png
# e.g., public/data/latest/tiles/5/22/28.png

# Check permissions
chmod -R 755 public/data/latest/tiles/
```

### Issue: Stale data showing

**Symptoms**: Dashboard shows old dates

**Diagnosis**:
```bash
# Check metadata
cat public/data/latest/metadata.json

# Check timestamp
jq '.generatedAt' public/data/latest/metadata.json
```

**Solutions**:
```bash
# Hard refresh browser (clears local cache)
# Cmd/Ctrl + Shift + R

# Or clear browser cache manually

# If still stale, regenerate:
python scripts/process_minimal.py
python scripts/generate_tiles_v3.py
npm run build
npm run deploy
```

### Issue: Grid queries return null (hover not working)

**Symptoms**: Hovering over map shows "No data" everywhere

**Diagnosis**:
```bash
# Check grids.json exists and has data
```
